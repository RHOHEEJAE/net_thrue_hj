# net_thrue_01 — 프로젝트 동작·기능 요약

## 한 줄 요약

**웹/앱에서 발생하는 행동 이벤트(페이지뷰, 클릭, 스크롤 등)를 설정만으로 수집하고, Kafka·DB에 쌓아서 대시보드로 분석할 수 있는 자체 트래킹 서비스**입니다.

---

## 동작 방식

1. **관리자**가 웹 Admin에서 **변수·트리거·태그**를 편집하고 **배포**합니다.
2. 수집 대상 **웹 페이지**에 스크립트(`ntm.js`) 한 줄만 넣으면, SDK가 배포된 설정을 받아옵니다.
3. 사용자가 페이지를 보거나 클릭·스크롤하면 **트리거**가 발동하고, **변수** 값과 함께 이벤트 1건이 서버(`/collect`)로 전송됩니다.
4. 서버는 받은 이벤트를 **Kafka** 토픽으로 보내고, **컨슈머**가 Kafka에서 읽어 **PostgreSQL**에 원시 데이터·집계 데이터를 저장합니다.
5. **대시보드**에서 기간·페이지·채널 등으로 필터링해 **페이지별 이벤트**, **퍼널 이탈률**, **스크롤·클릭 히트맵**을 조회할 수 있습니다.

즉, **“언제(트리거), 무엇을(변수), 어떻게(태그) 수집할지”**를 코드 배포 없이 설정으로 바꿀 수 있고, 수집된 데이터는 **Kafka → DB → 대시보드** 흐름으로 활용됩니다.

---

## 주요 기능

| 구분 | 내용 |
|------|------|
| **수집** | 페이지뷰, 클릭, 스크롤(25/50/75/99%), 커스텀 이벤트 등. 변수·트리거·태그로 조합·필터링 가능. |
| **설정·배포** | Admin에서 변수/트리거/태그 편집 → “게시”로 현재 설정 반영. 배포 이력·롤백 지원. |
| **행동 분석** | 페이지별 이벤트 건수, 퍼널 단계별 진입/이탈, 스크롤·클릭 히트맵. 기간·필터 조회. |
| **계정·권한** | 관리자(admin) / 일반 사용자(user). 가입 후 관리자 승인 필요. 관리자만 설정·배포·계정 관리. |
| **인프라** | FastAPI, PostgreSQL, Kafka, Zookeeper. Docker Compose 한 번에 기동. |

---

## 사용자 입장에서 보면

- **관리자**: 로그인 → 컨테이너 관리(변수/트리거/태그 편집, 배포) → 행동 분석 대시보드 조회 → 계정 관리(가입 승인·권한 지정). 필요 시 Kafka UI에서 수집 토픽(test) 메시지 확인.
- **일반 사용자**: 로그인 → 행동 분석 대시보드에서만 기간·필터로 데이터 조회.
- **개발자/운영**: 수집 대상 페이지에 `ntm.js` 로드 한 줄 추가. 설정 변경은 Admin에서만 하면 되고, 코드 배포 없이 수집 항목을 바꿀 수 있음.

---

## 기술 구성 (요약)

- **백엔드**: FastAPI (설정 API, 수집 API, 로그인·계정·대시보드 API)
- **클라이언트**: ntm.js (설정 수신, 트리거 리스너, 변수 계산, 이벤트 전송)
- **저장**: PostgreSQL(계정·원시 이벤트·집계 테이블), 파일(data/*.json, settings/latest.json)
- **스트리밍**: Kafka(test 토픽) — 수집 이벤트 실시간 전달, 컨슈머가 DB 적재·집계
- **실행**: Docker Compose(Postgres, Zookeeper, Kafka, API, 컨슈머, 퍼널 집계, Kafka UI)
